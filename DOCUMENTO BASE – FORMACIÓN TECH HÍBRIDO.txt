CONTEXTO GENERAL DEL PROYECTO

El proyecto Formacion Tech Híbrido nace con un objetivo operativo concreto:

Diseñar e implementar un sistema simple y robusto para la captura de paros de producción por célula, causa y duración, con el fin de generar métricas confiables que soporten decisiones de mantenimiento y mejora continua.

Principios rectores:

* No alterar la operación real de planta
* No interferir con mantenimiento ni PLCs
* Priorizar claridad y robustez sobre complejidad
* Operar como piloto antes de cualquier escalamiento

============================================

ARQUITECTURA GENERAL DEL SISTEMA

El sistema está dividido en capas claramente separadas:

1. Captura (Frontend)
2. Backend mínimo (puente de datos)
3. Persistencia segura (histórico)
4. Análisis y métricas
5. Automatización de salida

Cada capa tiene una única responsabilidad y no invade a las demás.

PUNTO BLOQUEADO ACTUALMENTE

El envío automático de correos depende de la disponibilidad de un servidor SMTP corporativo.

Situación actual:

* La red permite salida a internet
* El puerto SMTP público (Gmail 587) está bloqueado
* Es necesario definir SMTP corporativo (Exchange / relay interno)

Acción pendiente:

* Confirmación por parte de IT del servidor SMTP disponible
* Host, puerto y tipo de autenticación

Este bloqueo no afecta la captura de datos ni las métricas.
